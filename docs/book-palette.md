# Book Tools palette button catalog

Every button in the **Wolfram Book Tools** palette (`File > Palettes >
Wolfram Book Tools`, installed from
<https://www.wolfram-media.com/resources/> as the `WolframBookTools`
paclet, source at
`~/Library/Wolfram/Paclets/Repository/WolframBookTools-<version>/`) with
its visible label, the package function it calls, and the proposed
`MarkdownToNotebook` equivalent (markdown source, frontmatter key, or
`#|` cell option). Status: **[done]**, **[partial]**, **[todo]**.

The Book Tools paclet is a different beast from the Documentation Tools
palette covered in [palette.md](palette.md):

| | Documentation Tools | Book Tools |
|---|---|---|
| Output | one **page** per notebook (Symbol / Guide / TechNote / Overview), part of a paclet's `Documentation/` | one **chapter** per notebook, with a separate `Contents.nb` and `References.nb`, in a flat **Chapters Directory** |
| Stylesheet | per-page-type sheet (`FunctionPageStylesExt`, …), built into the System | `BookToolsStyles.nb` shipped by the paclet (book-oriented styles: `Chapter`, `SolvedExample`, `Theorem`, `Proof`, `Vocabulary*`, `Exercise*`, `QASection`, `Reference`, `MoreExplore`, …) |
| Build pipeline | `DocumentationBuild` turns authoring nbs into deployed reference pages | print-ready PDFs (`WBTPreviewPDF`) and Kindle reflow (`ExportForKindle`); no automated web deploy |
| Cross-doc links | resolved `paclet:Pub/Pkg/(ref\|guide\|tutorial)/...` URIs | cell-tag links (`WBTInsertLinkCellTag` + `WBTInsertLinkToSectionCell`), counter-box reference numbers, manual `paclet:` links |
| Project manifest | `PacletInfo.wl` | the Chapters Directory itself (no manifest); per-notebook `CounterAssignments -> {{"Section", n-1}}` |

The proposal in this file is a new `Template: Chapter` for
`MarkdownToNotebook`, plus the directory-level conventions to author a
whole book from a folder of `.md` files.

## The palette at a glance

The palette has one main panel (no tabs) and one sub-palette
(`Production Tools`). Buttons are inserted at the cursor / on the
selection of the *most recently used input notebook*, just like the
Documentation Tools palette.

## Proposed `Template: Chapter` for `MarkdownToNotebook`

The new template:

```yaml
---
Template: Chapter
Name: "Working with Plots"        # chapter title (used in the Section heading)
ChapterNumber: 3                  # the CounterAssignments value (Section, n-1)
Subtitle: "From ListPlot to Plot3D"   # optional; renders as Subchapter
PageWidth: 720                    # optional; sets PageWidth on the notebook
ShowPageBreaks: true              # optional
StyleSheet: "BookToolsStyles.nb"  # the default for Template: Chapter
---
```

is rendered by a new `chapterNotebook[data]` clause in
`MarkdownToNotebook.wl` (registered in the template-dispatch
`Switch` at the same place as `symbolNotebook`, `guideNotebook`,
`tutorialNotebook`, `overviewNotebook`).

The output is `Notebook[cells, StyleDefinitions ->
FrontEnd`FileName[{"Wolfram"}, "BookToolsStyles.nb", CharacterEncoding ->
"WindowsANSI"]]`, with cells assembled exactly the way `WBTChapterTemplate[]`
ships them in `WolframBookToolsTemplate.nb`:

1. **Chapter heading** (`Section` style, with `CounterAssignments`) — built from the H1 + `ChapterNumber:` frontmatter.
2. **Body** — H2/H3/H4/H5 become `Subsection`/`Subsubsection`/`Subsubsubsection`/`Subsubsubsubsection`; prose paragraphs are `Text`; bulleted lists are `Item`/`Subitem`/`Subsubitem`; ordered lists are `ItemNumbered`; ` ```wl ` blocks are `Input` (cached `Output` paired); a single prose line directly above an `Input` becomes `CodeText`.
3. **Special-cased headings** — H2s with reserved titles drop into the named back-matter section style:
    - `## Summary` → `SummarySection` (paragraphs → `SummaryNote`, bullets → `SummaryList`)
    - `## Vocabulary` → `VocabularySection` (definition-list → `VocabularyTable`)
    - `## Exercises` → `ExerciseSection` (numbered items → `Exercise`; `**Solution.**` → `ExerciseSolution`)
    - `## Q&A` → `QASection` (Q. / A. lead-ins)
    - `## Tech Notes` → `TechNoteSection`
    - `## More to Explore` → `MoreExploreSection`
    - `## Resources` → `ResourcesSubsection`
    - `## Takeaways` → `TakeawaysSection`
    - `## References` → `ReferenceSection` (or use the book-level `References.md`)
4. **Block constructs** as Pandoc-style fenced divs:
    - `::: solved-example` … `:::` → `SolvedExample` + body + `SolvedExampleEndCap`
    - `::: theorem` … `:::` → `Theorem(Numbered)` + `TheoremStatement` + `ProofTheoremDisplay…` + `ProofTheoremEndCap`
    - `::: proof` … `:::` → `Proof` + `ProofContent`
5. **Cell options** (`#|` directives), all new:
    - `#| flag: print-only | web-only | tentative | comment | lockdown` — secondary cell style
    - `#| page-break-above: true|false|automatic|inherited` / `#| page-break-below: …`
    - `#| margins: { left: N, right: N, top: N, bottom: N }`
    - `#| font-tracking: -0.1`
    - `#| line-spacing: 1.2`
    - `#| image-size: 200` (on `wl` cells; resizes graphics in the output)
    - `#| caption: "…"` / `#| captions: ["a", "b", "c"]` (drops `GraphicsEmbeddedCaption` rows)
    - `#| show-cell-label: false`
    - `#| close: true` (the reverse-close `"Close"` cell tag)
    - `#| anchor: my-anchor` (cell-tag link target, mirrors `WBTInsertLinkCellTag`)
    - `#| index: ["term1", "term2"]` (mirrors `WBTInsertIndexField`)
6. **Inline runs** — same as the existing inline grammar (` `code` ` → `InlineFormula`, ` ``code`` ` → `InlineCode`, `$math$` → `InlineMath`, `[text](url)` → `Hyperlink` / `Link` depending on scheme) plus:
    - `<kbd>X</kbd>` → `Key0`/`Key1`/`Key2`/`Key3` (sized by string length, same rule the palette uses)
    - `[text](path/to/chapter.md#anchor)` → `ButtonBox` resolving to `NotebookOpen + NotebookFind` against the `#anchor` cell tag (resolved at build time, not runtime)
    - `[@cite-key]` → `CounterBox`-linked `ButtonBox` resolving to the `Ref<n>` cell tag in the book-level `References.nb`

### Book-level build

A new `examples/Book/build.wls` walks a directory of `chapter-*.md` (plus
optional `references.md`), runs `MarkdownToNotebook` on each into a
sibling `Chapters/` directory, then emits `Contents.nb` (mirroring
`WBTGenerateContentsDialog` -> `WBTMakeContentsFromDialog`) and, if a
`references.md` is present, `References.nb` (mirroring
`MakeReferencesNotebook`). The output directory is exactly what `Set
Chapters Directory` would pick — so a user can open the palette, point it
at the build output, and edit interactively.

### What the palette does that markdown won't have

Out of scope for the markdown layer (author-time / front-end only):

- **Track-changes (Suggestion / Stet)** — author-time review workflow. Use git diff on the markdown source.
- **Annotation insertion / navigation** — analogous to the Documentation Tools `AnnotationInsert` / `AnnotationSearch`: not represented (the doc has no `TooltipBox` annotation cells).
- **Nudge L/R/Up/Down** — formula-box subscript / superscript fine-tuning. Author the formula correctly instead.
- **Kindle reflowable export** — separate pipeline; the markdown source is the source of truth, and pandoc / a converter pass should produce the Kindle output rather than re-using the Book Tools exporter.
- **PDF link rewriting (`WBTFixLinksforPDFs`)** and **System-link-to-web rewriting (`WBTConvertSystemLinksToWebLinks`)** — both are post-processing passes on the assembled `Chapters/*.nb`; surface as build-script flags rather than markdown features.
- **Defective-link inspector** — `wolframscript build.wls --check-links` (a build-time pass over the markdown).

## Things to verify before implementation

- The `CharacterEncoding` of the stylesheet reference is `"WindowsANSI"` in `CreateNewChapter` but `"UTF-8"` elsewhere; match what the paclet ships so a saved chapter opens the same way on Windows and macOS.
- `CounterAssignments` on the `Section` cell needs to reset `Subsection`/`Subsubsection`/`Exercise` to 0 in addition to `Section -> n-1` — the palette does this in `WBTSectionWithCounterAssignment`; chapters created via the New Chapter dialog only reset `Section`.
- The `Subsubsection` style has a `SubsubsectionTopMargin` variant for cases where it's preceded by a `Subsection` — the build needs to apply that variant by position.
- `MathLink``CallFrontEnd[FrontEnd``NotebookGetLayoutInformationPacket[…]]` (used by `WBTMakeContentsFromDialog` to get page numbers) only works with an interactive front end. The build-script equivalent of `Contents.nb` with page numbers requires a `UsingFrontEnd[…]` block.
